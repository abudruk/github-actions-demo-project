OneFuse XUI Documentation


The OneFuse XUI relies on the OneFuse Python Plugin, which is installed during setup.

OneFuse XUI Setup


  1. 1. Once the OneFuse XUI is installed in CloudBolt via UI Extensions, the onefuse directory should be at:

    • /var/opt/cloudbolt/proserv/xui/onefuse

  2. 2. Restart Apache from each CloudBolt Appliance

    • systemctl restart httpd

  3. 3. Create a Connection Info for OneFuse.

    • *Add OneFuse in Labels field.

  4. 4. Add OneFuse Parameters to CloudBolt Blueprints as shown below.


OneFuse Parameters


Quick Start list of all OneFuse Parameters to execute OneFuse Modules that can be applied to CloudBolt Environments, Blueprints, Groups, etc.

Usage Format is 'name' : 'value_format', replacing { string } with the actual value.


OneFuse ModuleParameter Name and Value Format
Ansible Tower 'OneFuse_AnsibleTowerPolicy_{executionstring}_{customId}' : '{onefusehost}:{policyname}:{hosts}:{limit}'

Property Toolkit

Static Property Sets 'OneFuse_SPS_{customId}' : '{staticpropertysetname}'
Create Properties 'OneFuse_CreateProperties_{customId}' : '{"key":"{key}","value":"{value}"}'

Valid Execution Strings

The following is a list of valid {executionstring} options for OneFuse Modules Ansible Tower and Scripting, and Pluggable Modules:

Execution StringNotes
HostnameOverwrite Execute OneFuse Module during hostname overwrite.
PreCreateResource Execute OneFuse Module prior to creating the resource.
PreApplication Only valid when using native CloudBolt Configuration Manager - will execute OneFuse Module prior to the Configuration Manager running.
PostProvision Execute OneFuse Module after provisioning the resource.
